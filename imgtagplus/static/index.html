How to use ImgTagPlus

A quick guide to tagging your images

Welcome to ImgTagPlus. It uses locally run models to automatically assign tags to your images without requiring internet access.

1. Directory Path

Provide an absolute path to the directory containing your images. E.g., /Users/name/Pictures/Vacation. ImgTagPlus will automatically skip files it cannot process and will look for standard image types.

2. Models

  • CLIP (Zero-Shot): The default, lightning-fast model. It compares images to a list of over 600 hardcoded tags and selects the best matches. Uses minimal RAM.
  • Florence-2: A rich Small Vision Language Model. It doesn't pick from a list; it actually "looks" at the picture and writes a detailed caption, then extracts tags from that caption. Excellent for identifying rare objects, text, and scene context. Requires a slightly more powerful machine.

3. Tags & Thresholds

Max Tags controls the maximum amount of keywords added to the XMP sidecar file.

Confidence Threshold (Applies to CLIP only): The lower the number, the more lenient the model will be with its guesses. A threshold of 0.25 is recommended.

4. XMP Files

ImgTagPlus is non-destructive. It writes .xmp sidecar files directly next to your original images. Open these in apps like Lightroom, Bridge, digiKam, or Darktable to see your embedded keywords instantly.

System Performance Ratings

Estimated speed per model based on your hardware

Excellent GPU Native (MPS or VRAM > 8GB)
  • CLIP: ~0.1s per image
  • Florence-2 Base: ~0.5s - 1s per image
  • Florence-2 Large: ~1s - 2s per image
Good Optimized CPU (RAM > 16GB)
  • CLIP: ~1s per image
  • Florence-2 Base: ~2s - 4s per image
  • Florence-2 Large: ~5s - 8s per image
Poor Heavy CPU (Low RAM)
  • CLIP: ~3s per image
  • Florence-2 Base: ~10s - 15s+ per image
  • Florence-2 Large: Not recommended

System Accelerator

How ImgTagPlus uses your hardware

The Accelerator tells ImgTagPlus what hardware to use for computing the models.

  • cuda: Dedicated Nvidia GPUs with VRAM. Extremely fast tagging.
  • mps: Apple Silicon (M1/M2/M3). Uses unified memory. Extremely fast.
  • cpu: The central processor. No dedicated hardware was found. Tagging will be significantly slower.
